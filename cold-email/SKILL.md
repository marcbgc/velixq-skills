## Workflow - Always follow these steps

1. Visit the website - analyze it for specific weaknesses
2. Identify the ONE biggest problem (mobile, Google, outdated, broken, no photos)
3. Find the contact email and name if visible
4. Write the cold email using the formula below
5. Output the email in a clean code block (see Output Rules below)

---

## Cold Email Formula (5-part structure)

### Part 1 - HOOK
Claim the new website is already built. Creates instant curiosity.
"Ich habe mir erlaubt, eine neue Version Ihrer Website zu erstellen - kostenlos, damit Sie sehen was moeglich ist."

### Part 2 - PAIN
Name the ONE specific problem found on their website. Be concrete, not generic.
Examples:
- "Ihre Website ist auf dem Handy kaum nutzbar"
- "Bei Google sind Sie fuer '[ihre Leistung] Muenchen' kaum sichtbar"
- "Ihre Website wurde seit Jahren nicht aktualisiert und wirkt dadurch unprofessionell"
- "Ihre Website ist nicht erreichbar"

### Part 3 - COMPETITOR THREAT (ALWAYS include - never skip)
This is the most important psychological trigger for Handwerker.
"Das bedeutet konkret: Wer heute in [Stadt] nach "[Leistung]" sucht, findet Ihre Mitbewerber - nicht Sie. Diese Kunden sind weg, bevor Sie ueberhaupt wissen dass sie existieren."

### Part 4 - CREDIBILITY
"Ich bin Marc Begic von Velixq Digital aus Muenchen. Ich spezialisiere mich auf Handwerksbetriebe und habe bereits fuer mehrere Sanitaer- und Heizungsbetriebe in Muenchen moderne Websites gebaut - mit dem Ergebnis: mehr Anrufe, mehr Anfragen, mehr Auftraege."

### Part 5 - CTA
Always a Zoom-Call, 15 minutes, no pressure.
"Ihre neue Website ist fertig. Ich wuerde sie Ihnen gerne in einem kurzen Zoom-Call zeigen - 15 Minuten, kein Druck, keine Verpflichtung. Wann passt es Ihnen diese Woche?"

## Output Rules (CRITICAL - always follow exactly)

RULE 1: Always output the subject line in its own code block first
RULE 2: Always output the full email in a single fenced code block using triple backticks
RULE 3: Preserve ALL line breaks and blank lines exactly as written - never collapse paragraphs
RULE 4: Never add markdown formatting inside the code block (no bold, no bullets, no headers)
RULE 5: After the code block, add a short 3-bullet analysis of the specific problems found

Ich habe Ihre neue Website fertig - darf ich sie Ihnen zeigen?

## Rules

- ALWAYS visit the website before writing - never write generic emails
- ALWAYS include the competitor threat (Part 3) - never skip it
- ALWAYS claim the website is already built - creates urgency and curiosity
- ALWAYS output in a code block so the user can copy it directly without editing
- NEVER send on weekends or Friday afternoon
- NEVER use more than 6 sentences in the main body
- NEVER use bold, bullets or markdown formatting inside the email code block
